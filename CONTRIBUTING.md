# 🤝 Contributing to GitFlow AI

感谢您对 GitFlow AI 的兴趣！我们欢迎所有形式的贡献。

## 如何贡献

### 报告 Bug

1. 使用 GitHub Issues 提交问题
2. 使用清晰的标题描述问题
3. 提供复现步骤
4. 提供环境信息（OS、Python版本等）
5. 附上错误日志

### 提交功能请求

1. 使用 GitHub Issues 提交
2. 描述功能的用例
3. 解释为什么这个功能对项目有价值

### 提交代码

1. Fork 本仓库
2. 创建特性分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'feat: add amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 创建 Pull Request

## 开发规范

### 代码风格

- 遵循 PEP 8 规范
- 使用 Black 进行代码格式化
- 使用类型注解

### 提交信息规范

遵循 Conventional Commits：

- `feat`: 新功能
- `fix`: 修复
- `docs`: 文档
- `style`: 格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 维护

### 测试

- 为新功能添加测试
- 确保所有测试通过

## 联系方式

如有问题，欢迎通过 GitHub Issues 联系我们。
